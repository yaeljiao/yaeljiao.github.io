Node.js之NODE_ENV

发表于 2017-06-20

​ 在Node.js项目中NODE_ENV用来表明当前的工作环境（开发环境或生产环境），通过在代码里判断NODE_ENV这来对生产环境和开发环境做不同的处理，比如在开发环境中添加日志功能、mongodb数据库的操作等。

环境配置的几种方式

​ 在Linux环境中，分别通过编辑/etc/profile和~/.bash_profile来设置系统环境变量和当前用户的环境变量，在配置文件中添加export NODE_ENV="development"从而配置成开发环境，再执行source ~/.bash_profile使配置立即生效。

​ 若要配置一次性的环境变量，在Linux & mac bash中执行export NODE_ENV=production，在Windows中执行set NODE_ENV=production。
